# Future SQLite Model

The secure base starts with files and JSONL. SQLite should be added later when run history, search, memory, and dashboards need indexing.

Future tables:

- `tasks`
- `runs`
- `plans`
- `steps`
- `tool_events`
- `safety_events`
- `artifacts`
- `knowledge_sources`
- `memories`
- `lessons`
- `cert_reports`

SQLite should be a read/index layer at first, not the only source of evidence.
