# S.E.R.A. Personal and Professional V1 Roadmap

This roadmap locks the planned path after the 16-milestone Base MVP. It is planning-only. It does not implement additional Milestone 13 behavior, Milestones 17 through 24, iOS, browser automation, external providers, new SQLite tables beyond the Base MVP line, update systems, backup systems, enterprise operation, Hive Mode, robotics, or product functionality.

The machine-readable source of this lock is `architecture/post-base-mvp-roadmap-v1.json`.

## Completion Definitions

Base MVP completion remains Milestone 16: Portable Offline Base MVP Acceptance.

Personal and Professional V1 completion is Milestone 24: S.E.R.A. Personal and Professional V1 Release.

Milestone 24 is not the permanent end of S.E.R.A. development. After Milestone 24, future work is capability, device, domain, enterprise, robotics, distributed-system, and autonomy expansion rather than completion of missing foundational product promises.

After Milestone 24, we choose what S.E.R.A. learns next.

## Product Guarantee

S.E.R.A. Personal and Professional V1 is a locally controlled, nontechnical, desktop-and-mobile-supervised intelligence operating system capable of ingesting real materials, using certified local models, performing multiple certified professional workflows, using the public web only under governance, evaluating work, preserving evidence, learning from corrections, preventing certified recurring failures, improving capabilities, and producing professional delivery packages.

## Explicit Non-Guarantees

Milestone 24 does not prove universal expertise, zero mistakes, every professional domain, unattended business authority, unrestricted client communication, unrestricted account access, autonomous payments or contracts, complete Runtime operation entirely on iPhone, enterprise multi-user deployment, distributed Hive Mode, robotics, unrestricted high-risk professional advice, or permanent completion of all future S.E.R.A. development.

## Milestones 17-24

Milestone 17 - Real-World Pilot and Defect Burn-Down: planned. Requires real operator projects, actual local models, real documents and knowledge libraries, long Studio sessions, interruption and recovery, corrections, failure and improvement evidence, recurrence signals, defect prioritization, and release-blocking defect resolution.

Milestone 18 - Nontechnical UX, Reliability, Updates, and Recovery: planned. Requires guided installation, first-run setup, hardware and model readiness, local-service startup, plain-language health, updates, rollback, backup, restore, storage management, diagnostics, crash recovery, accessibility, and no terminal requirement for ordinary use.

Milestone 19 - Real Local Intelligence and Multimodal Runtime: planned. Requires certified bounded support for actual local general, reasoning, coding, vision-capable, and retrieval or embedding models where justified, plus PDF extraction, OCR, image analysis, audio transcription, selected video processing, model routing, hardware-aware resource policy, and media provenance.

Milestone 20 - Multi-Studio Professional Capability Portfolio: planned. Requires Evidence and Research Studio, Website Studio, Business Documentation Studio, Data and Reporting Studio, and one additional selected professional Studio, each with exact versions, capabilities, operator workflow, evaluation, revision, evidence, delivery packaging, learning signals, limitations, and rollback readiness.

Milestone 21 - Professional Services and Fiverr Operations: planned. Requires client intake, requirements clarification, scope and deliverable definition, estimate assistance, project workspaces, revision limits, quality gates, delivery checklists, export packages, portfolio evidence, reusable service templates, client-data separation, and human approval before external communication or delivery. It is not unattended Fiverr-account automation.

Milestone 22 - Governed Web, Browser, and External Tool Runtime: planned. Requires per-task network authorization, browser isolation, domain approval, download verification, upload approval, form-action review, source capture, website preview and testing, governed publishing, API connector contracts, credential isolation, public-network evidence, cancellation, revocation, and no silent browsing.

Milestone 23 - Secure Mobile Gateway and iOS Companion: planned. Requires secure device pairing, local-network operation, optional encrypted remote operation, revocable device credentials, request submission, file/image/audio/camera intake, approvals, progress, notifications, evidence and deliverable review, revision requests, cached offline client state, audit, and lost-device revocation. The complete Runtime remains on the desktop or privately controlled host.

Milestone 24 - S.E.R.A. Personal and Professional V1 Release: planned. Requires nontechnical installation without Git or Codex, normal Desktop Operator launch, actual local-model operation, core offline operation, governed optional web access, multiple certified professional Studios, professional delivery packages, recurrence-prevention demonstration, persistence across restart, backup and restore, update and rollback, secure iOS pairing, iPhone request and supervision, approval protection for high-risk actions, no terminal requirement for ordinary use, one real Fiverr-style project from intake to delivery package, visible capability and limitation matrix, defined soak test, clean installation and uninstall, and a distributable versioned release.

## Boundaries

The iOS companion is a secure supervision and request-submission client. The complete Runtime stays on the desktop or privately controlled host.

Fiverr readiness means professional service workflows, delivery packages, and human-approved external communication. It does not mean unattended account automation, payments, contracts, or unrestricted client contact.

Governed web access means per-task authorization, domain approval, isolated browsing, source capture, upload/form review, credential isolation, cancellation, and revocation. Silent browsing remains forbidden.

Multimodal support means certified bounded local processors and provenance. It does not mean unrestricted media understanding or trusted model claims.

Nontechnical-user readiness means ordinary operation without terminal use, with guided install, first-run setup, health, update, rollback, backup, restore, diagnostics, and accessibility surfaces.

Learning and recurrence prevention by Milestone 24 must demonstrate prevention of certified recurring failures within declared scope. It does not imply zero mistakes or universal prevention.

## Future Expansion

After Milestone 24, S.E.R.A. can continue through capability expansion, device expansion, domain expansion, enterprise expansion, robotics expansion, distributed-system expansion, and autonomy expansion.
