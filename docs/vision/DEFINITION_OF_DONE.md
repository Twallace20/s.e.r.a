# Definition of Done

A S.E.R.A. feature is done only when all of the following are true:

- The feature has a written contract.
- The feature has a clear package owner.
- The feature has safety boundaries.
- The feature writes artifacts or structured evidence.
- The feature has tests or certs.
- The feature reports honest status.
- Failure modes are documented.
- Runtime data is ignored or intentionally exported.
- No placeholder implementation is counted as complete.

Valid statuses:

- `completed`
- `completed_with_changes`
- `no_op`
- `blocked`
- `failed`
- `needs_approval`
