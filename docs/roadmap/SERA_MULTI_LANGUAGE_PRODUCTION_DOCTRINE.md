# S.E.R.A. Multi-Language Production Doctrine

## Principle

S.E.R.A. should not be limited to Python projects. Python is one production lane, not the whole production system.

The mature system should route each project to the most useful language, framework, runtime, and validation path for the job. S.E.R.A. should learn the domain, build a sandbox, practice safely, validate results, and then use the language stack that fits the deliverable.

## Required Language Families

- **TypeScript** — web apps, operator consoles, agents, typed integrations, SaaS frontends, Node tooling.
- **JavaScript** — websites, browser automations, Node utilities, platform scripting.
- **Python** — AI workflows, data tools, automation, research systems, prototypes, dashboards.
- **Swift** — iOS, macOS, SwiftUI, Apple-platform app production.
- **Kotlin** — Android apps, JVM services, mobile prototypes.
- **Dart** — Flutter cross-platform apps.
- **Java** — enterprise services, Android legacy, JVM backends.
- **C#** — Unity games, .NET apps, Windows tools.
- **C++** — Unreal projects, performance systems, robotics simulation, graphics, game prototypes.
- **C** — embedded systems, microcontrollers, safe electronics labs.
- **Rust** — safe systems tools, CLIs, performance-sensitive services.
- **Go** — services, infrastructure tools, CLIs, concurrent backends.
- **SQL** — data models, reporting, dashboards, migrations, analytics.
- **HTML/CSS** — websites, landing pages, UI prototypes, templates.
- **PHP** — WordPress and legacy CMS work.
- **Ruby** — Rails and legacy web apps.
- **PowerShell** — Windows automation and approved local worker operations.
- **Bash** — Unix automation, CI scripts, and local tooling.

## Sandbox Rule

For every language family, S.E.R.A. should use the Sandbox Learning Doctrine:

```text
Study → notes → safe practice task → sandbox attempt → evidence → refinement → retry → validation → approval.
```

## Commercial Rule

S.E.R.A. should sell or deliver only language capabilities that have passed practice tasks, QA checks, and evidence review. New languages can be learned, but they should graduate through sandbox proof before being used on serious client work.
