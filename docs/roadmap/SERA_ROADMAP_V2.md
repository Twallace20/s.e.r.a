# S.E.R.A. Roadmap v2

Status: Active roadmap truth / owner-review aligned.

S.E.R.A. is no longer planned as a command-prompt-only project. The roadmap is now organized around a local-first operator system with multiple surfaces and a universal engine.

## North star

S.E.R.A. should become a universal, local-first, self-improving operator that can ingest sources, learn domains, practice, evaluate, iterate, use approved tools, build deliverables, create proof, improve herself through approved branches, and operate under owner authority within legal and safe bounds.

## Roadmap lanes

1. **Execution Spine** — safe command boundaries, approved command execution, approved branch development, rollback, validation, and evidence.
2. **Operator Control Plane** — dashboard, approval inbox, evidence viewer, project queue, blocker visibility, status, and operator decisions.
3. **Universal Engine** — source ingest, domain learning packs, practice tasks, rubrics, evaluation, iteration, and deliverable generation.
4. **Website + Service Surface** — public service site, portfolio, case studies, proof, intake, pricing, and client trust layer.
5. **Mobile/App Companion** — mobile web/PWA approvals first, then optional native app after the workflow proves useful.
6. **Prime-User Agency Engine** — service packages, proposals, delivery workflows, proof, analytics, and revenue operations.
7. **Productization Decision Path** — only after S.E.R.A. proves useful privately, decide whether to keep her private, sell Lite/Studio, or build a platform.

## Phase 101–150 — Universal Engine

Phases 101–120 build universal ingest. Phases 121–150 build universal deliverables. This is the heart of S.E.R.A. entering a domain through the same learn, practice, evaluate, revise, and deliver loop.

## Phase 151–200 — Private Studio, Website, Mobile Companion, and Universal Operator

Phases 151–160 make the private studio useful. Phases 161–170 convert the existing site path into a public service/proof/intake site. Phases 181–190 build a mobile web/PWA approval companion before native app work. Phase 200 is Universal Operator Alpha.

## Phase 201–300 — App, agency engine, and productization

Phases 201–225 move toward a native app only after the mobile web/PWA proves useful. Phases 226–250 optimize prime-user agency mode. Phases 251–300 are optional productization and platform readiness.

## Autonomy rule

More autonomy only comes with more evidence, more reversibility, more approval control, and better operator visibility.

## Away-mode rule

Away mode may plan, propose, branch, edit within approved scope, validate, capture proof, and prepare packets. Away mode must not self-merge, self-deploy, purchase, post externally, access secrets, expand scope, or make legal, medical, financial, or licensed professional decisions.
